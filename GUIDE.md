## 1. Node.js 설치하기

Node.js는 이 프로그램을 실행하는 데 필요한 기본 프로그램이에요.

### 1-1. Node.js 다운로드

1. 브라우저를 열고 주소창에 입력하세요:
   ```
   https://nodejs.org/ko
   ```

2. 사이트에 들어가면 두 개의 다운로드 버튼이 보여요
   - **왼쪽 초록색 버튼 (LTS)** ← 이걸 클릭하세요!
   - 오른쪽은 최신 버전인데, 우리는 안정적인 LTS를 사용할 거예요

3. 다운로드된 파일을 실행하세요 (예: `node-v20.x.x-x64.msi`)

### 1-2. Node.js 설치

1. 설치 마법사가 열리면 **"Next"** 클릭
2. 라이선스 동의 체크하고 **"Next"** 클릭
3. 설치 경로는 그대로 두고 **"Next"** 클릭
4. 구성 요소도 그대로 두고 **"Next"** 클릭
5. **"Install"** 클릭해서 설치 시작
6. 설치 완료되면 **"Finish"** 클릭

### 1-3. 설치 확인하기

설치가 잘 됐는지 확인해볼게요!

1. **Windows 키**를 누르고 `PowerShell` 검색
2. **Windows PowerShell** 클릭해서 실행
3. 검은색(또는 파란색) 창이 열리면, 아래 명령어를 입력하고 Enter:
   ```
   node --version
   ```
4. `v20.x.x` 같은 버전 숫자가 나오면 성공! 🎉

> ⚠️ 만약 "'node'은(는) 내부 또는 외부 명령..." 이런 오류가 나오면, 컴퓨터를 재시작한 후 다시 해보세요.

## 2. 프로젝트 다운로드

두 가지 방법 중 편한 걸 선택하세요!

---

### 방법 A: ZIP 다운로드 (추천! 가장 쉬움)

1. 아래 링크를 클릭해서 ZIP 파일 다운로드:
   
   👉 https://github.com/Daewooki/naver-bc-automation/archive/refs/heads/main.zip

2. 다운로드된 ZIP 파일을 **바탕화면**에 압축 해제
   - ZIP 파일 우클릭 → "압축 풀기" 또는 "여기에 풀기"

3. 압축 해제하면 `naver-bc-automation-main` 폴더가 생겨요
   - 폴더 이름을 `naver-bc-automation`으로 변경해주세요 (선택)

4. PowerShell을 열고 해당 폴더로 이동:
   ```
   cd "$env:USERPROFILE\Desktop\naver-bc-automation-main"
   ```
   
   > ⚠️ 폴더 이름을 변경했다면:
   > ```
   > cd "$env:USERPROFILE\Desktop\naver-bc-automation"
   > ```

---

### 방법 B: Git 사용 (개발자용)

> ⚠️ Git이 설치되어 있어야 해요! 
> 설치 안 되어 있으면 **방법 A**를 사용하세요.

1. PowerShell을 열고 바탕화면으로 이동:
   ```
   cd "$env:USERPROFILE\Desktop"
   ```

2. 프로젝트 다운로드:
   ```
   git clone https://github.com/Daewooki/naver-bc-automation.git
   ```

3. 폴더로 이동:
   ```
   cd naver-bc-automation
   ```

## 3. 패키지 설치하기

이 프로젝트가 사용하는 라이브러리들을 설치할 거예요.

### 3-1. 기본 패키지 설치

아래 명령어를 입력하고 Enter:

```
npm install
```

이 과정은 **1~3분** 정도 걸려요. 여러 줄의 텍스트가 주르륵 나오는데, 정상이에요!

기다리면 이런 메시지가 나와요:
```
added xxx packages in xxs
```

### 3-2. 브라우저 자동화 도구 설치

자동으로 브라우저를 조작하기 위한 도구를 설치해요.

```
npx playwright install chromium
```

이것도 **1~2분** 걸려요. 크롬 브라우저를 다운로드하는 거예요.

완료되면 이런 메시지가 나와요:
```
Downloading Chromium xxx ... done
```

## 6. API 키 설정

가장 중요한 단계예요! 집중해주세요 👀

> 💡 **OpenAI** 또는 **Gemini** 중 하나만 선택하면 돼요!
> - OpenAI: 유료 (크레딧 필요)
> - Gemini: **무료!** ⭐ (구글 계정만 있으면 OK)

---

### 6-1. 옵션 A: Gemini API 키 발급받기 (무료! 추천 ⭐)

**무료**로 사용할 수 있어서 처음 시작하기 좋아요!

1. 브라우저에서 이 주소로 접속:
   ```
   https://aistudio.google.com/app/apikey
   ```

2. **구글 계정**으로 로그인하세요

3. **"API 키 만들기"** 버튼 클릭

4. 프로젝트 선택하라고 나오면 아무거나 선택 (또는 새로 만들기)

5. `AIza` 로 시작하는 긴 키가 나와요. **지금 바로 복사하세요!**

> ⚠️ **중요!** 이 키는 한 번만 보여줘요. 반드시 메모장에 저장해두세요!

---

### 6-2. 옵션 B: OpenAI API 키 발급받기 (유료)

GPT-5.2 모델을 사용하고 싶다면 이 방법을 선택하세요.

1. 브라우저에서 이 주소로 접속:
   ```
   https://platform.openai.com/api-keys
   ```

2. 로그인하세요 (구글 계정으로 가능)

3. 처음이라면 계정 설정을 해야 해요:
   - 전화번호 인증
   - 결제 수단 등록 (신용카드)
   - **크레딧 충전** (최소 $5 정도 권장)

4. API Keys 페이지에서 **"Create new secret key"** 버튼 클릭

5. 이름은 아무거나 입력 (예: "블로그자동화")

6. **"Create secret key"** 클릭

7. `sk-` 로 시작하는 긴 키가 나와요. **지금 바로 복사하세요!**

> ⚠️ **매우 중요!** 이 키는 한 번만 보여줘요. 창을 닫으면 다시 볼 수 없어요!
> 반드시 메모장이나 안전한 곳에 저장해두세요.

### 6-3. 네이버 블로그 ID 확인하기

1. 네이버 블로그에 로그인
2. 내 블로그 주소를 확인하세요
3. 주소가 `https://blog.naver.com/abc123` 이라면
4. **abc123** 이 내 블로그 ID예요!

### 6-4. .env 파일 수정하기

1. 바탕화면의 `naver-bc-automation` 폴더를 열어요

2. `.env` 파일을 찾아요
   > ⚠️ 파일이 안 보인다면? 
   > 탐색기 상단 메뉴에서 "보기" → "숨긴 항목" 체크!

3. `.env` 파일을 **메모장**으로 열어요
   - 파일 우클릭 → 연결 프로그램 → 메모장

4. 파일 내용을 아래처럼 수정해요:

**🅰️ Gemini 사용 시 (무료!):**
```
# AI 선택 (gemini 또는 openai)
AI_PROVIDER=gemini

# Gemini API 키 (무료!)
GEMINI_API_KEY=AIza여기에아까복사한키붙여넣기

# 네이버 블로그 ID (필수)
NAVER_BLOG_ID=내블로그아이디

# 데이터베이스 (이건 그대로 두세요!)
DATABASE_URL="file:./prisma/dev.db"
```

**🅱️ OpenAI 사용 시 (유료):**
```
# AI 선택 (gemini 또는 openai)
AI_PROVIDER=openai

# OpenAI API 키 (유료)
OPENAI_API_KEY=sk-여기에아까복사한키붙여넣기

# 네이버 블로그 ID (필수)
NAVER_BLOG_ID=내블로그아이디

# 데이터베이스 (이건 그대로 두세요!)
DATABASE_URL="file:./prisma/dev.db"
```

**Gemini 예시:**
```
AI_PROVIDER=gemini
GEMINI_API_KEY=AIzaSyABC123abc123ABC123abc123
NAVER_BLOG_ID=happyblogger123
DATABASE_URL="file:./prisma/dev.db"
```

5. **Ctrl + S** 눌러서 저장하고 메모장 닫기

> ⚠️ **절대 주의!** 
> - API 키 앞뒤로 따옴표 붙이지 마세요
> - 공백(스페이스) 넣지 마세요
> - 이 파일을 다른 사람과 공유하지 마세요!

## 7. 네이버 로그인

자동 발행을 위해 네이버에 로그인해둘 거예요.

### 7-1. 로그인 명령 실행

PowerShell에서:

```
npm run login
```

### 7-2. 브라우저에서 로그인

1. 자동으로 크롬 브라우저가 열려요
2. 네이버 로그인 페이지가 나와요
3. **평소처럼 로그인하세요!**
   - 아이디/비밀번호 입력
   - 또는 QR 로그인
   - 2단계 인증이 있다면 그것도 진행

4. 로그인 성공하면 PowerShell에 이런 메시지가 나와요:
```
✅ 세션이 저장되었습니다!
🎉 이제 자동화가 이 세션을 사용합니다.
```

5. 브라우저가 자동으로 닫혀요

> 💡 세션은 보통 7~30일간 유지돼요. 나중에 발행 실패하면 다시 로그인하면 돼요!

## 9. 사용 방법

### 9-1. 브랜드커넥트 링크 추가하기

1. 브랜드커넥트에서 받은 링크를 복사해요
   - 예: `https://naver.me/ABC123xyz`

2. 대시보드의 URL 입력창에 붙여넣기

3. **"추가"** 버튼 클릭

4. 리스트에 링크가 추가돼요!

### 9-2. 블로그 글 발행하기

1. **🚀 발행** 버튼 클릭!

2. 확인 창에서 "확인" 클릭

3. 브라우저가 자동으로 열리고 작업이 시작돼요:
   - 상품 페이지 접속 & 이미지 수집
   - GPT가 리뷰 글 작성
   - 블로그 에디터 열기
   - 제목 입력
   - 이미지 업로드
   - 본문 입력
   - 해시태그 추가
   - 발행 버튼 클릭!

4. 모든 과정이 자동으로 진행돼요. 약 1~2분 걸려요.

5. 발행 완료되면 대시보드에 **"발행완료"** 표시!

6. **"📄 발행된 글 보기"** 링크로 확인해보세요!

### 9-4. 여러 개 발행하기

- 링크 여러 개 추가해두고 하나씩 발행하면 돼요
- **⚠️ 주의:** 한 번에 너무 많이 발행하지 마세요!
- **권장:** 하루 1~3개 정도

## 10. 문제 해결

### ❌ "node를 찾을 수 없습니다" 오류

**원인:** Node.js 설치 후 터미널을 다시 시작 안 함

**해결:**
1. PowerShell 창을 닫고
2. 컴퓨터 재시작
3. 다시 PowerShell 열고 시도

---

### ❌ "npm install" 에서 오류

**해결:**
1. 아래 명령어로 캐시 정리:
   ```
   npm cache clean --force
   ```
2. 다시 `npm install` 실행

---

### ❌ 네이버 로그인이 안 돼요

**원인:** Playwright 브라우저 문제

**해결:**
```
npx playwright install chromium --force
```
그 다음 다시 `npm run login`

---

### ❌ 발행이 실패해요

**원인 1:** 세션 만료

**해결:**
```
npm run login
```
다시 로그인 후 발행

**원인 2:** 네이버 점검 중이거나 네트워크 문제

**해결:** 잠시 후 다시 시도

---

### ❌ OpenAI API 오류

**원인 1:** API 키가 잘못됨

**해결:** .env 파일에서 API 키 확인
- 앞뒤 공백 없는지
- 따옴표로 감싸지 않았는지
- 전체 키를 다 복사했는지

**원인 2:** 크레딧 부족

**해결:** [OpenAI 결제 페이지](https://platform.openai.com/account/billing)에서 크레딧 충전

---

### ❌ "localhost:3000 연결 거부"

**원인:** 서버가 실행 중이 아님

**해결:**
1. PowerShell에서 `npm run dev` 실행 중인지 확인
2. 실행 중이 아니면 다시 실행
3. "Ready" 메시지가 나온 후 접속

---

### ❌ 이미지가 안 올라가요

**원인:** 상품 페이지 구조 변경 또는 네트워크 문제

**해결:**
1. 상품 페이지가 정상인지 직접 확인
2. 잠시 후 다시 시도
3. 계속 안 되면 다른 상품으로 테스트
